# 校园租车管理系统

**个人全栈项目** | Spring Boot 3.5.9 + MyBatis-Plus + MySQL + WebSocket + Spring Security + JWT

## 项目概述

面向高校场景的多品类车辆租赁平台，支持汽车、摩托车、电动自行车、自行车等车辆类型的在线租赁，覆盖从选车估价、下单支付、取还车核验到评价的完整业务闭环，同时提供管理端仪表盘、实时客服、公告系统等运营功能。数据库
20+ 张表，后端 285+ 个 Java 文件，独立设计开发。

## 技术亮点

- **定价引擎**
  ：抽象费用项（FeeItem）与优惠项（DiscountItem）模型，支持基础租金、押金、增值服务按天计费，结合用户角色权益（折扣率、免押比例、免费延长次数）动态计算实付金额，下单时对价格、角色快照固化，防止后续调价影响历史订单
- **订单状态机**：设计 8
  种状态（待支付→已支付→待取车→租赁中→待还车→已完成/已取消/异常），每步状态流转附带前置校验（车辆状态、用户并发订单上限、延长次数限制），还车环节支持多罚金项录入（车损/超时/清洁等），自动从押金抵扣，余额退回用户账户
- **实时客服**：基于 STOMP over WebSocket 实现一对一会话，用户发起对话自动分配管理员，JWT 握手拦截器校验身份，消息支持文本/图片/文件/语音四种类型；利用
  MySQL JSON 列存储会话参与者，通过 JSON_CONTAINS 查询匹配会话
- **异步操作日志**：自定义 @OperLog 注解 + AOP 环绕通知，将日志写入剥离到单线程线程池（容量 1000
  的有界队列），避免阻塞主业务；注解支持指定模块、日志级别、操作描述，配合全局异常处理器统一返回 Result 结构
- **无状态认证体系**：Spring Security + JWT（jjwt 0.12.3），OncePerRequestFilter 拦截所有请求，解析 Bearer Token 注入
  SecurityContext；Token 包含 userId + role，配合 token 校验接口供前端判断登录态；支持 Token 刷新

## 功能模块

| 模块   | 说明                                                                                                                       |
|------|--------------------------------------------------------------------------------------------------------------------------|
| 用户端  | 注册登录（图形验证码）、车辆浏览/筛选、费用预估、下单支付（余额/微信/支付宝）、申请取还车、延长租期、订单列表/详情、个人信息管理、学生认证、评价晒图点赞、留言反馈                                      |
| 管理端  | 仪表盘（核心指标、30 天订单与收入趋势、车辆出租率）、车辆与车型 CRUD、订单全流程处理（确认取车、核验还车含罚金、取消/异常标记）、用户管理与角色权益配置、公告发布（支持定时、附件、分类）、网点管理（含经纬度）、维修记录、系统日志查询 |
| 基础设施 | 文件上传（按年月日分目录）、全局异常统一封装、MyBatis-Plus 自动填充（创建/更新时间、逻辑状态）、分页插件、Druid 连接池、跨域 CORS 配置、学生身份到期定时降级（@Scheduled）                  |

## 数据库设计（部分表）

`car_info`（车辆档案，含 VIN/里程/油量/保险到期/年检到期/日租金/押金/状态等 17 字段）、`car_models`
（车型模板，品牌车系/能源类型/排放标准/变速箱/驱动方式/座位数等 19 字段）、`order`（订单主表，含价格快照/角色快照/时间/金额/罚金
22 字段 + 2 外键）、`order_log`（操作流水）、`order_penalty`（罚金明细）、`order_service`（订单增值服务）、`user`
（含余额/积分/累计消费/学生到期日/驾驶证/身份证）、`role_benefits`（角色权益配置：押金比例/租金折扣/免费延长次数/超时费率）、
`review` / `review_like`（评价与点赞）、`service`（增值服务）、`station`（取还车网点，含经纬度）、`notice` / `notice_attachment`
（系统公告与附件）、`maintenance_record`（维修记录）、`conversation` / `message`（实时聊天）
